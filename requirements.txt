# faster-whisper (ctranslate2 backend) is the PRIMARY STT path and runs
# large-v3-turbo on CUDA at ~15x real-time. ctranslate2 ships its own CUDA
# runtime but needs the cublas + cudnn DLLs from these nvidia-* wheels, which

# ── PyTorch (CUDA build for the RTX 3090) ─────────────────────────────────
# REQUIRED for GPU acceleration of: RAG reranker (sentence-transformers
# CrossEncoder), long-term-memory embedder, voice-id (resemblyzer), and
# wake-word VAD (silero). Python 3.14 has NO torch+cuda wheel on PyPI, so it
# MUST be installed from the PyTorch CUDA index or these all silently run on
# CPU (5-20x slower). The 2026-05-30 audit found torch was 2.12.0+cpu; this
# is the cu126 build that put them back on the 3090. The --extra-index-url
# adds the PyTorch CUDA index ALONGSIDE PyPI, so the cu126 torch wheel resolves
# here while every other package still comes from PyPI. (A plain
# `pip install torch` pulls the CPU wheel and undoes the GPU win.)
--extra-index-url https://download.pytorch.org/whl/cu126
torch==2.10.0+cu126

# Standby auto-engage lyric-detection loop (skills/standby_audio_detect.py).
# OPTIONAL. Runs whisper-tiny on a 3 s rolling mic buffer every 5 s and uses
# librosa onset_strength (spectral flux) + a rhyme-density heuristic to
# decide whether the mic is hearing vocal music. After 15 consecutive
# seconds of positive detection AND the headset being the active output,
# JARVIS auto-engages standby and TTS "I'll wait until you call, sir".
# Without librosa the loop disables itself silently — the rest of the skill
# (spectral classifier + wake-word gate) keeps working. Uncomment to enable:
# librosa

# Real-time mic cleanup (core/audio_processor.py — noise-cancel-1). OPTIONAL.
# The processor applies echo cancellation → noise suppression → AGC to every
# captured chunk before it reaches Whisper, so JARVIS can hear past TTS
# playback, fans, HVAC, keyboard noise and distant speakers. Each layer
# probes for a preferred backend and falls back to a numpy implementation
# when missing, so leaving these uncommented is safe — they only enhance
# quality if installed. Set AUDIO_PROCESSING_ENABLED=False in
# bobert_companion.py to bypass the pipeline entirely.
# Uncomment to install:
# noisereduce              # spectral-subtraction noise suppression (NS layer)
# webrtc-audio-processing  # WebRTC APM — AEC3 + NS + AGC in one binding
                           # (Windows wheels are spotty; the processor
                           # falls back to numpy AEC/NS/AGC when missing)
